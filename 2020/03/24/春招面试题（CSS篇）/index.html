春招面试题（CSS篇）

说说CSS选择器以它们的优先级

  • !important
  • 内联样式（1000）
  • ID选择器（0100）
  • 类选择器/伪类选择器/属性选择器（0010）
  • 元素选择器/伪元素选择器/关系选择器（0001）
  • 通配符选择器（0000）

你知道BFC吗？

  • BFC是什么？

    BFC 全称为块级格式化上下文 (Block Formatting Context) 。BFC是 W3C CSS 2.1 规范中的一个概念，它决定了元素如何对其内容进行定位以及与其他元素的关系和相互作用，当涉及到可视化布局的时候，Block Formatting Context提供了一个环境，HTML元素在这个环境中按照一定规则进行布局。一个环境中的元素不会影响到其它环境中的布局。比如浮动元素会形成BFC，浮动元素内部子元素的主要受该浮动元素影响，两个浮动元素之间是互不影响的。这里有点类似一个BFC就是一个独立的行政单位的意思。可以说BFC就是一个作用范围，把它理解成是一个独立的容器，并且这个容器里box的布局与这个容器外的box毫不相干。

  • BFC触发的条件有哪些？

  1. float不为none
  2. position不为relative和static
  3. overflow为auto、scroll、hidden
  4. display的值为table-cell或inline-block
  5. flex或inline-flex

  • BFC可以干什么？

    1. 解决浮动元素领父元素高度坍塌的问题

    当对 未设置高度的父元素 中的所有子元素设为浮动时，就会导致父元素的高度坍塌。


    解决方法：
    1. 对父元添加 overflow: hidden 或 display: table-ceil 等属性;
    2. 使父元素也浮动，这种方法的缺点是，会影响后续元素的排列;
    3. 为浮动的最后一个元素设置伪元素::after {clear: both};

      2. 解决两栏布局左侧宽度固定，右侧自适应宽度
